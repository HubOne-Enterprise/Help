New Items in this Release

 

#7679 Ability to Search on any part of the client list

This feature adds to Mail, Templates and Mail Templates and means when you are manually searching for a

client, you can type any part of the client name and the application will filter based on any part of the client

name.

#6841 Single Sign-On for CCH

Instructions have been produced showing how to configure and establish single sign on for CCH iFirm in

OnePractice.

#7674 Add Custom Field Popups when user clicks send in Mail Templates

Previously, mail templates would add only data from practice management to emails. Now when you click

send, you can add custom data. This has been delivered in this release.

#7678 Terms and Conditions

Terms and Conditions were amended to support GDPR and change location of HubOne Support Website.

#6809 Templates Re-Design

Templates has a new, modern look and feel.

#6737 Add Documentation on Archive Library

Documentation Added

#7666 Scans: Add Ability to discard and Delete Saved Items

Version 6.2 Released 5th March 2018

Welcome to the Release Notes for OnePractice 6.2. This release was authorised on 16 February 2018. This document contains information on the release, and also details fixes included in this release.

 

In this document:

•New items in this release

•Breaking changes

•Items resolved in this release

New Items in this Release

 

#6847 CCH Provisioning

The production provisioning platform was modified to include features specific for CCH iFirm Document Manager. Which includes:

•The Wolters Kluwer provisioning option

•Ability to add CCH homepage icons (also relates to #6381)

•Ability to create a link to a customer's acclipse url from the homepage

•Green branding applied to the provisioned SharePoint environment

•Updated sample templates

 

#6848 Create OCR Solution for Scans

The team have worked on a new project, Scan Doctor. The Scan Doctor app is designed to perform Optical Character Recognition of PDF documents prior to them being consumed and processed by OnePractice Scans. A number of customers have reported that they have a number of PDF files, or regularly generate non-searchable PDF files, which Scans cannot process. Scan Doctor is able to convert these files to Searchable PDF files for OnePractice Scans processing. This solution has been developed and is currently in Beta for testing by partners prior to a slated release in version 6.3.

 

#6982 CCH Update for Partners

HubOne has signed an agreement with Wolters Kluwer to white label OnePractice as a new product, CCH iFirm Document Manager. This offers a great opportunity for partners to offer Office 365 licencing and support services for CCH iFirm customers that require this.

 

#6086 Provisioning Wizard Guide

The guidance for the Provisioning Wizard has been updated.

 

#6381 Creation of new OnePractice Icons for CCH

New icons for the OnePractice Portal Page have been developed to work closely with the Wolters Kluwer look and feel.

 

#6732 OnePractice Guides Online

HubOne has made available our new documentation library, developed for version 6.1. It is available at https://software.hubone.com/help.

 

#6923 Scans Upgrade to Include Other Name

Documents coming from various sources include other names for individuals, stopping Scans matching correctly. Scans now include the other name field.

 

#5741 Upgrade of HubOne Software Site

The HubOne Software Site at https://software.hubone.com has undergone a significant upgrade.

 

#6301 OnePractice Mail - Add Auto-refresh functionality

OnePractice Mail now includes auto-refresh functionality for refreshing client lists based on the schedule defined in common app settings.

 

#6775 Provisioning - Change to new UI Paradigm

The Provisioning application was difficult to see in extremely high resolution environments. Provisioning has been rewritten to support these common new devices.

 

#6870 Single Sign on for CCH iFirm

We have added documentation to enable single sign in for CCH iFirm using Azure Active Directory Enterprise Applications features.

 

#4542 Scans - Allow for Client Exclusion

There is a new feature in Scans which allows the exclusion of specific clients from the application. This is also stored in the settings.

 

#6302 OnePractice Scans - Add Auto-refresh functionality

OnePractice Scans now includes auto-refresh functionality for refreshing client lists based on the schedule defined in common app settings.

Known Issues

 

#6784 Amend Help Font for Mobile Help

The help font on mobiles is not standard.

WORKAROUND: None

 

#6785 Amend Help Font for CCH Mobile Help

The help font for CCH Mobile is not standard

WORKAROUND: None

 

#6995 Scan Doctor Crashes after >60 threads

If Scan Doctor starts too many threads, it enters a race condition.

WORKAROUND: Use the Thread setting to reduce the number of threads used

 

#6960 Templates: App gets stuck on 1st use for the day

There have been a small number of reports (less than 5) of templates not refreshing on 1st use.

WORKAROUND: Wait 2 minutes and refresh the browser

 

#6975 Scans: When using SharePoint or OneDrive as scans location, no popup

When using SharePoint or OneDrive as the Scans folder for Scans, the manual popup may not appear

WORKAROUND: Only use a standard local folder as your scans location

 

#6240 Mail: OnePractice Category appears white

There have been a small number of reports (less than 2) of the Mail category appearing white

WORKAROUND: Open Master Categories and add the category manually

 

#6527 Migration Manager using legacy licensing

This does not affect end users or partners

WORKAROUND: None

 

#6656 Templates: Stuck Refreshing lists

There have been a small number of reports of templates getting stuck refreshing lists. We are investigating

WORKAROUND: Wait 2 minutes and refresh the browser

 

#6157 SharePoint: Need a method to list and checkin all checked out files in large library

In a large library there is no simple method to list and checkin all checked out files. This is a SharePoint Limitation. We are investigating a solution.

WORKAROUND: None

 

#4808 SharePoint Search

Clients are reporting difficulty using SharePoint Search, Whilst some is user education, use of metadata eases this problem.

WORKAROUND: Use Metadata to filter documents even in large lists. Ensure indexing is enabled. Contact us if you need assistance with large lists

 

#6353 Document Center OneDrive Permissions Error

Using Files on Demand in Windows 10 Build 1709, some users are reporting issues having read-write access to OneDrive

WORKAROUND: Remove Mandatory MetaData and Check in/ Check Out

 

#6418 Migration Manager takes significant time to clean.

On Large document sets with limited RAM and Processor, Migration Manager takes significant time to clean files

WORKAROUND: Run on a large Azure Server
